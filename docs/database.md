# vegaRead Database Documentation

## Overview

vegaRead uses **SQLite** as its embedded database, accessed through the **sqlx** async Rust library. The database file is stored in the platform-specific Tauri application data directory and is initialised automatically on first launch.

---

## Technology

- **SQLite** — embedded, serverless, file-based relational database
- **sqlx** — async Rust library for type-safe SQL queries
- **SqlitePool** — a connection pool injected into the Tauri app state and shared across all command handlers via Tauri's **State<SqlitePool>** mechanism

---

## Schema

### vagaread table

One row per imported EPUB book.

| Column | Type | Constraints | Description |
|---|---|---|---|
| vagaread_id | TEXT | PRIMARY KEY | UUID v4, generated at import time |
| internal_fp | TEXT | NOT NULL | Absolute path to the EPUB copy inside the app data directory |
| meta_data | TEXT | NOT NULL | Full EPUB metadata serialised as a JSON string |
| current_read_idx | INTEGER | NOT NULL, DEFAULT 0 | Character offset within the current spine item (reading progress) |
| current_spine | INTEGER | NOT NULL, DEFAULT 0 | Spine index of the last read chapter (reading progress) |
| is_deleted | BOOLEAN | NOT NULL, DEFAULT 0 | Soft-delete flag; deleted books are hidden from the library but not removed from disk |

---

## Database Module (db.rs)

All SQL operations are implemented as async functions in **src-tauri/src/db.rs**. No raw SQL is written in handlers or other modules.

### create_record(pool, record)

**Used by:** upload_file_handler (via load_file_core)

Inserts a new vagaread row. The vagaread_id is a UUID generated by the handler before calling this function. meta_data is serialised from serde_json::Value to a string.

### list_all_vb_records(pool)

**Used by:** show_home_page_handler

Returns all vagaread rows where is_deleted = false. The meta_data column is deserialised back into serde_json::Value before the record is returned.

### get_vb_record_by_id(pool, id)

**Used by:** get_ebook_content_handler, list_spine_handler

Fetches a single vagaread row by vagaread_id (passed as a String). Returns an ApplicationError with code DATABASE_ERROR if no record is found.

The internal_fp from this record is used to re-open the EPUB file for content extraction and spine listing. The database is the single source of truth for where the file lives on disk.

### update_vb_record (planned)

**Not yet called by any active handler.**

Will update current_spine and current_read_idx for a given vagaread_id. Intended to be called whenever the reader advances to a new chapter or page, so that reading progress is persisted across sessions.

## Data Flow: Metadata Storage

When an EPUB is imported:
1. **epub_util::extract_epub_metadata** extracts all metadata properties from the EPUB into a **HashMap<String, Vec<String>>**.
2. The map is serialised to a JSON string (e.g. {"title":["My Book"],"creator":["Author Name"]}).
3. This JSON string is stored in the **meta_data** column as TEXT.
4. On read, sqlx deserialises it into **serde_json::Value**, which Tauri serialises back to JSON for the frontend.
5. The frontend receives it as a plain JavaScript object and accesses it through the **BookMetadata** TypeScript interface.

Because the metadata schema is stored as a free-form JSON blob rather than individual columns, new EPUB metadata properties are automatically preserved without requiring a schema migration.

---

## Connection Pool Lifecycle

The **SqlitePool** is created in **lib.rs** during app setup and registered as Tauri managed state:

```
.manage(pool)
```

Each **#[tauri::command]** function receives it as **pool: State<'_, SqlitePool>** and passes **pool.inner()** (a reference to the underlying pool) to the db functions.

The pool is kept open for the lifetime of the application process and closed automatically when the app exits.

---

## File Storage

Imported EPUB files are copied into the Tauri **app_data_dir** by **util::copy_to_app_directory**. The copy's absolute path is stored in **internal_fp**. The original file at the source path is not modified or deleted.

Storing a copy ensures that moving or deleting the original file does not break the library. The trade-off is additional disk usage.

## Future Work

- Implement the **update_vb_record** handler call on every chapter/page navigation to persist reading progress.
- Add an index on **is_deleted** if the library grows large enough to benefit from it.
- Consider adding a **date_added** column (TEXT ISO-8601) for sorting the library by import date.
- Consider a **cover_cache_fp** column to store the path to an extracted cover image once cover extraction is implemented.
